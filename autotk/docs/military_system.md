# ⚔️ 汉末三国「部众矩阵与军备演进」宏观设计规范

本规范确立了《三国演兵录》兵卒体系底层重构的设计。废除原有“现代兵种分类”，采用 **“基础兵科 × 护甲制式 × 兵刃远射”** 的三维矩阵，动态生成符合典籍历史的部众称谓与战斗属性向量。

---

## 一、 核心元素之「三维矩阵抽象」

每个城池驻扎的部众均由以下三个维度的配置组合而成：

### 1. 基础兵科 (Core Branch) ── 兵卒的战术骨架
*   `SWD` (短兵/刀盾系)：近身先锋与阵线掩护。
*   `LNC` (长兵/矛戟系)：阵地防守与列阵御骑。
*   `RNG` (远射/弓弩系)：远程齐射火力压制。
*   `CAV` (突骑/骑兵系)：野战突破与地缘奔袭。

### 2. 护甲制式 (Armor Class) ── 影响防御与负重移速
*   `ARM_NONE` (无甲/布衣)：基础防御（防御力 +0%），最高速度。
*   `ARM_LEATHER` (犀皮/革甲)：轻装护甲（防御力 +25%），无移速惩罚。
*   `ARM_扎甲` (扎甲/铁铠)：重装铁甲（防御力 +60%），轻微降低强攻机动力。
*   `ARM_具装` (战马具装)：人马合一铁铠（仅骑兵，防御力 +120%），大幅降低机动，但极大增加冲撞伤害。

### 3. 兵刃与远射制式 (Weapon System) ── 决定攻击属性与战术克制
*   `WEP_STANDARD` (标配武器)：无特殊加成。
*   `WEP_鋼铤` (百炼钢铤)：攻击力 +50%，具备“破甲”属性（无视防守方郡县繁荣度城墙加成）。
*   `WEP_劈山斧` (重劈宽刃)：攻击力 +40%，具备“劈山”属性（克制短兵刀盾防御）。
*   `WEP_大黄弩` (强力擘张弩)：射程与攻击力极高，远程先头齐射阶段伤害 +80%。
*   `WEP_连弩` (元戎十矢连弩)：齐射战力 +30%，在首轮对射阶段可连续执行两次判定。
*   `WEP_马槊` (重型突骑槊)：骑兵专用，冲锋攻击力 +60%。

## 二、 称谓与属性融合生成引擎 (符合历史的扩展命名)

系统会根据这三个核心维度的配置组合，在 UI 渲染和日志中动态解析出极具历史沉浸感的**融合称谓**：

```
[基础兵科] + [护甲制式] + [兵刃远射] => 动态历史称谓 (Name) & 战力属性向量 (Stats)
```

### 1. 短兵系 (`SWD`) 历史称谓映射
*   `SWD` + `ARM_NONE` + `WEP_STANDARD` $\rightarrow$ **【轻装刀手】**
*   `SWD` + `ARM_LEATHER` + `WEP_STANDARD` $\rightarrow$ **【环首刀手】**（汉军标配短兵）
*   `SWD` + `ARM_扎甲` + `WEP_STANDARD` $\rightarrow$ **【扎甲先锋】**
*   `SWD` + `ARM_扎甲` + `WEP_劈山斧` $\rightarrow$ **【重甲破阵士】**（《三国志》：“陷阵摧锋，勇举破阵”）
*   `SWD` + `ARM_LEATHER` + `WEP_劈山斧` $\rightarrow$ **【白刃解烦卫】**（孙权“解烦督”私兵典故）

### 2. 长兵系 (`LNC`) 历史称谓映射
*   `LNC` + `ARM_NONE` + `WEP_STANDARD` $\rightarrow$ **【持矛役从】**
*   `LNC` + `ARM_LEATHER` + `WEP_STANDARD` $\rightarrow$ **【列阵矛士】**
*   `LNC` + `ARM_扎甲` + `WEP_STANDARD` $\rightarrow$ **【持戟甲卫】**
*   `LNC` + `ARM_扎甲` + `WEP_鋼铤` $\rightarrow$ **【百炼破甲士】**（《作铤教》精铁破甲兵）
*   `LNC` + `ARM_扎甲` + `WEP_劈山斧` $\rightarrow$ **【积弩戟盾兵】**（戟楯拒骑之劲卒）

### 3. 远射系 (`RNG`) 历史称谓映射
*   `RNG` + `ARM_NONE` + `WEP_STANDARD` $\rightarrow$ **【散地弓手】**
*   `RNG` + `ARM_LEATHER` + `WEP_STANDARD` $\rightarrow$ **【控弦弩手】**（汉军主流弩机射手）
*   `RNG` + `ARM_LEATHER` + `WEP_大黄弩` $\rightarrow$ **【大黄弩士】**（汉代北军王牌远程弩兵）
*   `RNG` + `ARM_扎甲` + `WEP_连弩` $\rightarrow$ **【元戎弩卫】**（诸葛连弩精锐队）

### 4. 骑兵系 (`CAV`) 历史称谓映射
*   `CAV` + `ARM_NONE` + `WEP_STANDARD` $\rightarrow$ **【游侠突骑】**
*   `CAV` + `ARM_LEATHER` + `WEP_STANDARD` $\rightarrow$ **【轻装骁骑】**
*   `CAV` + `ARM_扎甲` + `WEP_马槊` $\rightarrow$ **【槊血铁骑】**（执槊突川之重甲铁骑）
*   `CAV` + `ARM_具装` + `WEP_马槊` $\rightarrow$ **【重装虎豹骑】**（曹操王牌宿卫，冲突破坏力之最）

## 三、 兵农合一：黔首、役夫与部众的转换机制

重构后，人口从抽象总量划分为三个地缘层级，通过征发与解甲实现循环：

### 1. 黔首 (平民百姓 - `civilians`)
*   **特征**：郡县税收与日常劳作的主体。黔首根据郡县繁荣度（`value`）每回合进行自然人口增长，持续提供钱粮和税收产出。

### 2. 役夫/役卒 (预备役/劳役 - `auxiliary`)
*   **特征**：已征发的青壮年预备力量。**不直接产生税收，且每回合消耗少许屯粮**。
*   **核心功能（修筑与补充）**：
    *   **补充兵员的唯一前置**：专业部众不能直接凭空招募。必须通过消耗本地的“役夫”，配发制式装备，将其“简拔”为对应的一阶新卒部众。
    *   **兴修基建的主力**：凡是“修筑农田水利”、“修葺加固城防”或“部署守城重器（筌蹄、木兽）”，除了耗费钱粮外，**必须优先指派并消耗本地的役夫**进行役作。

### 3. 三级转化指令
*   **【征发徭役】 (黔首 $\rightarrow$ 役夫)**：将普通平民青壮登记编册为役夫，扩充基建劳力与预备兵员（平民减少，役夫增加）。
*   **【简拔建制】 (役夫 $\rightarrow$ 部众)**：分发武器护甲，将役夫训练为一阶新卒部众（役夫减少，专业部众增加）。
*   **【解甲归田】 (部众 $\rightarrow$ 役夫/黔首)**：战争平息时，可将精锐部众裁撤还原为役夫或平民，恢复地方产出并停发日常军饷。

## 四、 战争军器 (守御/攻城重装)

*   **攻城重器**：
    *   **霹雳车 (抛石重器)**：降低防守方郡县繁荣度上限。
    *   **冲车 (撞门车)**：极大降低守御方的城墙防守加成。
*   **守城重器**：
    *   **清 (捕马铁夹)**：守城部署，彻底废除攻方突骑兵的冲撞攻击力加成。
    *   **木兽 (喷火器)**：守城部署，防守结算时消灭攻方大量轻装步卒部众。
