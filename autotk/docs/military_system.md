# ⚔️ 汉末三国「部众矩阵与军备演进」宏观设计规范

本规范确立了《三国演兵录》兵卒与人口体系底层重构的设计。废除原有“现代兵种分类”，采用 **“基础兵科 × 护甲制式 × 兵刃远射”** 的三维矩阵，并引入 **“黔首（平民） $\rightarrow$ 役夫（预备役） $\rightarrow$ 部众（正规军）”** 的汉代兵农合一与徭役转换机制。

---

## 一、 兵农合一：人口三大层级划分 (Demographics)

为了还原古代战争中徭役征发、伤亡折损与后勤生产的真实博弈，郡县（城池）的人口分为三个递进层次：

```
     黔首 (平民/日常税收)
           | 
           v (征发役夫 / 消耗资金)
     役夫 (劳役/预备役/消耗粮草)
      /    \
     /      \ (简拔编练 / 消耗装备)
    v        v
专业部众    兴修工程/军器
(四大兵科)   (消耗役夫折损)
```

### 1. 黔首 (平民/生民 ── `civilians`)
*   **古籍称谓**：**黔首**、**生民**、**百姓**。
*   **定位**：郡县税收与日常农业的基石。
*   **机制**：不直接参与战争，但每回合根据城池繁荣度自然增长。黔首数量直接决定了该郡县每回合的国库税金（贯）与粮食自然产出（石）。

### 2. 役夫 (徭役/预备兵/杂卒 ── `auxiliary`)
*   **古籍称谓**：**役夫**、**役卒**、**预备役**。
*   **定位**：基建劳力与正规军的唯一补充兵源。
*   **机制**：
    *   不产生税收，且每回合消耗一定的郡县屯粮。
    *   是所有军事简拔和城防建设的“原材料”。

### 3. 专业部众 (正规精锐 ── `garrison`)
*   **古籍称谓**：**部众**、**营伍**、**正规军**。
*   **定位**：已配发甲胄与兵刃、专司征战的职业军卒。
*   **包含兵科**：短兵（`SWD`）、长兵（`LNC`）、远射（`RNG`）、突骑（`CAV`）。

## 二、 役卒预备役核心政令与转化机制 (Conversion)

玩家与 AI 在郡县进行内政与军事操作时，必须通过以下政令打通人口流转：

### 1. 【征发徭役】 (黔首 $\rightarrow$ 役夫)
*   **政令内容**：消耗少许资金，强行将一定比例的“黔首”编入郡县“役夫”册籍。
*   **代价**：本城平民减少，日常税收和粮食自然产出略微下跌。

### 2. 【简拔编练】 (役夫 $\rightarrow$ 专业部众)
*   **政令内容**：**正规军的唯一来源**。消耗本地 $N$ 个“役夫”，并配发特定的护甲与武器，训练转化为一阶的新卒部众。
*   **代价**：扣除对应数量的役夫。

### 3. 【兴修大工】 (消耗役夫)
*   **政令内容**：进行郡县基建（如“修筑水利”、“加固关卡防线”）或督造防御重器（如“部署筌蹄”、“造木兽”）时。
*   **消耗**：必须调遣并消耗一定数量的本地“役夫”。在工程完工后，大部分役夫会“劳折”（死亡或伤残折损），小部分退役还原为平民。

### 4. 【解甲归田】 (部众 $\rightarrow$ 役夫/平民)
*   **政令内容**：战争平息或粮草吃紧时，可下令遣散部分精锐正规军，还原为“役夫”或直接降为“平民”，恢复郡县的税收与生产。
